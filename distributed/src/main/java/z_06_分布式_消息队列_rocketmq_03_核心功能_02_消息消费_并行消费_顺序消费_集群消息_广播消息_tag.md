#集群消息
```asp
- 每条消息只需要被处理一次，broker只会把消息发送给消费集群中的一个消费者
- 在消息重投时，不能保证路由到同一台机器上
- 消费状态由broker维护
```

#广播消息
```asp
- 消费进度由consumer维护

- 保证每个消费者消费一次消息

- 消费失败的消息不会重投

只有在消息模式为MessageModel.CLUSTERING集群模式时，Broker才会自动进行重试，广播消息不重试

```
